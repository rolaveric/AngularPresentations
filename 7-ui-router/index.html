I'm going to talk about Angular's URL routing system, and how ui-router improves on it.

URL Routing

Mapping a URL to a combination of:

  • Controller
  • Template
  • Dependency Providers

Just quickly for those that aren't aware, when I talk about routing I'm talking about this idea of the URL having a dynamic effect upon the application.
In the case of Angular, this means loading a new 'View' with it's own template and controller.

$route/$routeProvider short comings

  • Only 1 ng-view allowed
  • No hierarchy

So to recap, Angular's native routing is a great tool and most of the time, it's all we really need.
But it has a few things that stop it from being GREAT. The main one being the fact that you have only 1 ng-view to deal with.
Did you see how I had to use ng-include to build up the template to make use of previous templates?
Also, what if I wanted to create a sidebar or something? It can be done, but it starts to get... fiddly.
